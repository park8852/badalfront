# 바로배달 모의해킹 프로젝트 - 배경 및 필요성

## 📋 프로젝트 배경

### 1. 배달 서비스의 특성상 보안 취약점이 높은 이유

배달 앱은 다른 서비스와 달리 **개인정보가 매우 민감하고 상업적 가치가 높은** 서비스입니다.

#### 주요 취약점 요소

1. **개인정보의 다양성**
   - 주소 (집 주소, 직장 주소 등 실제 위치)
   - 전화번호 (스팸, 피싱 공격에 악용 가능)
   - 실시간 위치 정보
   - 결제 정보 (신용카드, 계좌번호)

2. **금전적 가치**
   - 포인트 시스템 (실제 돈과 유사한 가치)
   - 주문 결제
   - 정산 시스템 (가게 주인에게 전달되는 금액)
   - 환불 처리

3. **다중 사용자 역할**
   - 일반 사용자
   - 가게 주인 (사업자 정보 보관)
   - 관리자
   - 각 역할 간 권한 관리 복잡

4. **실시간 데이터 처리**
   - 주문 처리의 동시성 (Race Condition 발생 가능)
   - 위치 추적 실시간 업데이트
   - 결제 처리의 원자성 보장 필요

## 💡 왜 배달 앱을 모의해킹 대상으로 삼았는가?

### 1. 실무적 중요성

#### 개인정보 유출의 파급력
```
일반 쇼핑몰: 이메일, 이름만 유출
배달 앱: 주소, 전화번호, 결제정보, 실시간 위치 → 신체적 피해 가능
```

**예시:**
- 주소 유출 → 집 배달 공격 가능
- 전화번호 유출 → 스팸/피싱 공격
- 위치 정보 유출 → 실시간 추적 가능
- 결제 정보 유출 → 금전적 손실

### 2. 최신 트렌드 반영

#### 2024-2025년 주요 보안 트렌드

1. **Race Condition 취약점** (최신)
   - 게임, 이커머스에서 빈번히 발견
   - 배달 앱의 동시 주문 처리에 취약
   - 모바일 게임에서 보고된 사례 다수

2. **Secret Key 노출** (실무적 이슈)
   - GitHub에 AWS 자격증명 노출
   - CI/CD 파이프라인 취약점
   - 실제 보안 사고의 주요 원인

3. **GPS 메타데이터** (개인정보보호 이슈)
   - 소셜미디어의 EXIF 데이터 추출 문제
   - 배달 앱의 사진 업로드 기능에 위험

4. **JWT Algorithm Confusion** (2025 최신)
   - JWT 토큰 위조 공격
   - 최근 보고되는 인증 우회 기법

### 3. 교육적 가치

#### 다양한 보안 취약점 학습 가능
- SQL Injection
- XSS
- 파일 업로드 취약점
- Race Condition
- 비즈니스 로직 취약점
- 권한 관리 문제
